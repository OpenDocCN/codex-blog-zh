如果你像每年成千上万的人一样决定学习计算机科学，或者决定学习如何编码，那么你肯定会面临这个问题:我应该首先学习哪种编程语言？在大约 700 种不同编程语言的海洋中，我们能够将列表缩小到两种——Python 和 Ruby，这已经很了不起了。但是，你应该选择哪一个呢？

在本文中，我将讨论流行的编程语言 Python 和 Ruby 之间的一些关键区别。我也将给出我的观点，对于希望学习第一种编程语言的新程序员来说，哪种语言是更好的选择。

## 一般

Python 和 Ruby 都是通用的高级脚本语言。这意味着它们不需要像 C 和 C++等语言那样被编译，而是通过解释器传递。因此，Python 和 Ruby 比其他语言更容易学习和交互，这使它们成为编程语言的绝佳选择。

Python 是由荷兰数学家吉多·范·罗苏姆于 1991 年创造的。尽管是一种通用语言，Python 主要用于学术界、人工智能、机器学习和数据密集型编程。Ruby 是由一位名叫[松本幸宏](https://en.wikipedia.org/wiki/Yukihiro_Matsumoto)的日本计算机科学家在 1996 年创造的。Ruby 也是一种通用语言，主要用于 web 开发和函数式编程。

# 哲学

通常，编程语言的设计都带有某种哲学或理念。

Python 是建立在[Python 之禅](https://www.python.org/dev/peps/pep-0020/#id2)中阐述的原则之上的。Python 编程注重可读性和明确性，以确保代码易于从一个开发人员到另一个开发人员理解。由于这种透明性，用 Python 编写的代码往往比用其他语言编写的代码更容易调试。Python 试图通过确保只有一种解决问题的方法来突出简单性。因此，Python 在如何编写代码方面提供了很少的灵活性，而在程序员应该如何解决问题方面更加死板。

Ruby 的理念是围绕着编码中人的因素而构建的。Ruby 的设计理念是使用计算机为人类工作，而不是相反。因此，Ruby 代码通常类似于人类语言。因为 Ruby 以人类体验为中心，所以在如何解决问题方面非常灵活。Ruby 在开发者之间提供了巨大的灵活性，因为在 Ruby 中没有正确的做事方法。因为 Ruby 强调人与人之间的交互，所以它不像 Python 那样显式；由于这一点及其灵活性，Ruby 中的代码可能很难调试。在 Python 中，代码如何从一行流到下一行是非常清楚的，而 Ruby 在幕后发生了许多事情——想想 Ruby 的魔力吧！

# 结构

不讨论框架很难讨论 Python 和 Ruby。框架是用来帮助开发人员创建其他软件或应用程序的软件。它可以被认为是开发人员在创建自己的应用程序时使用的预制代码或工具。

Python 有 Django、Flask、CherryPy 等很多框架，其中 Django 是最流行的。 [Django](https://www.djangoproject.com/) 是一个支持 MVC(模型、视图、控制器)编程的 web 框架。众所周知，它具有可伸缩性，并且非常兼容不同的数据库和操作系统。

Ruby 主要以其 web 框架 Ruby on Rails (RoR)而闻名。 [Ruby on Rails](https://rubyonrails.org/) 是一个很好的 web 开发框架，因为它在开发过程中以速度著称。RoR 也因其社区和资源而闻名，如广泛的文档和教程视频。最后，有许多 Ruby Gems 可用于 Rails，允许快速、轻松地定制 web 应用程序。

# 社区

学习编码时，选择一门背后有庞大社区的语言是必不可少的。这将确保你可能会有许多问题的答案，以及语言的不断创新和发展。

因为 Python 是一种开源语言，许多人不断地、持续地在新项目上合作，同时也在创新我们所知道的 Python。由于它的不断变化，Python 社区正在不断地塑造语言的发展方向——你也可以！此外，由于 Python 被广泛应用于各种学科，其受欢迎程度也在稳步上升，Python 社区也在不断发展。因此，Python 程序员经常可以找到大量的资源来帮助他们完成最新的项目。此外，对于这样一个大型社区，库总是在更新和添加，给开发人员提供了越来越强大的工具。

像 Python 一样，Ruby 也是一种开源语言，这意味着它背后也有一个大型社区。然而，由于近年来受欢迎程度的下降，Ruby 社区没有 Python 社区那么大。通常，当谈到用 Ruby 编程时，是在 Rails 的上下文中，所以，这两个社区是可以互换的。Rails 社区，尽管比 Python 社区慢，也在不断创新 Ruby on Rails，并积极塑造 Rails 前进的方向。与其他框架相比，新的 gem 总是被添加和更新，这继续推动 Rails 向前发展。

# 职业机会

在决定先学哪种语言的时候，我相信很多人都会考虑可能的职业前景。

随着 Python 近年来人气暴涨，其工作机会也随之增加。当在 LinkedIn 上搜索 [Python 时，大约会返回 13 万个工作列表。这些工作包括从数据工程到网络开发。Python 是非常通用的，因为工作需要 Python 知识。尽管如此，由于 Python 开发人员拥有强大的库，许多工作似乎更倾向于涉及数据的角色。此外，](https://www.linkedin.com/jobs/search/?keywords=python)[根据 Glassdoor](https://www.glassdoor.com/Salaries/python-developer-salary-SRCH_KO0,16.htm) 的数据，Python 开发人员的平均年收入为 85，634 美元，与[软件工程师的平均年收入 103，807 美元](https://www.glassdoor.com/Salaries/software-engineer-salary-SRCH_KO0,17.htm)相比，这个数字可能偏低。一些使用 Python 的顶级公司包括:谷歌、脸书、Spotify、Dropbox 和网飞。

与 Python 不同，Ruby 的受欢迎程度近年来一直在缓慢下降。因此，工作机会也在减少，然而，仍然有许多公司雇佣 Ruby 开发人员。当在 LinkedIn 上搜索 Ruby 时，大约会返回 25000 个工作列表。这些职位大部分是面向 Ruby on Rails 开发人员和全栈工程师的。[根据 Glassdoor](https://www.glassdoor.com/Salaries/ruby-on-rails-developer-salary-SRCH_KO0,23.htm) 的数据，Ruby 开发人员的平均年收入为 78，878 美元，与[软件工程师的平均年收入 103，807 美元](https://www.glassdoor.com/Salaries/software-engineer-salary-SRCH_KO0,17.htm)相比，这个数字可能仍然偏低。一些使用 Ruby 的顶级公司包括:Airbnb、彭博、Github 和黄页。

# 选哪个？

Python 和 Ruby 都是很好的学习第一语言。然而，如果你决定为一家你知道正在积极使用 Ruby 的公司工作，那么我会建议你选择 Ruby。在所有其他情况下，我认为 Python 是明显的赢家。由于 Python 简单的语法和显式的本质，它的学习曲线肯定比 Ruby 短得多。此外，学好 Python 将使你更容易在语言之间切换，因为它更适用于不同的领域。虽然 Ruby 和 Python 都可以用于最基本的编程，但 Python 更适用，目前使用也更广泛。由于这些原因，我认为在选择第一种编程语言时，Python 将是您的最佳选择。

# 结论

最终，你决定先学习哪种语言并不重要。重要的是，你选择了一种语言并坚持下去。所有的编程语言都需要时间和奉献。无论你选择哪种语言，你都应该致力于发展一种亲密的关系。如果你学好了一门语言，再加上对计算机科学基础的理解，你应该能够在整个计算机科学世界中运用你的技能。

我把一个软件工程师在我考虑学习编码时给我的最好的建议留给你:“就挑一个，开始，不要回头。”

编码快乐！
