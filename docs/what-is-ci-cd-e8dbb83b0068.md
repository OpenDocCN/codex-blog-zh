# 什么是 CI/CD

## 这是一个在招聘广告中经常出现的术语。但这意味着什么呢？

CI/CD(持续集成和交付)这个术语最近风靡全球。

![](img/0b4317c42395701f5a4016f5cfa18e6a.png)

但这意味着什么呢？团队如何实现它，这种方法的好处是什么？这仅仅是工程经理为了打发时间而创造的另一个时髦词汇，还是有任何价值？是时候让这位科技领域最受欢迎的领袖来报道 HRs 最喜欢的新术语了。

# 要点

1.  **什么是 CI/CD-** 如上所述，CI/CD 代表*持续集成&交付。*它建立在一个简单的原则上，经常提交和合并你的代码变更。提交的代码通过一个自动化的工作流运行，以测试是否符合标准，如果是好的，就被接受。
2.  **CI-**这个过程的第一部分是 CI 或持续集成。顾名思义- **持续集成您对代码所做的任何更改。开发人员可能一天提交几次代码。这确保了一切总是更新的。假设我通过改变数据收集的方式来改变数据管道。从事转换数据的 API 工作的人应该确保他们的过程没有过时。不断集成新的变化增加了对此的保护。**
3.  如果有人不得不手工检查和合并每一个变更，事情将会失去控制。这就是 CD(连续交付)步骤的用武之地。在提交被完全接受之前，任何提交的更改都将通过一系列自动化的构建来运行。一旦提交被接受，它就会被快速集成到生产中。
4.  **CI/CD 的好处-** 这种方法有几个好处，但它们都可以归结为一个本质- *平滑。* CI/CD 使跨时区的大型团队工作变得更加容易。自动化测试的存在对于确保代码变更不会破坏任何东西是非常重要的。就其本质而言，CI/CD 中的变化非常小，因此检测 bugs 问题要容易得多。它还确保了所做的每一个更改都被考虑在内，因此在大型项目中，人们可能在没有相互交互的情况下处理相关的组件，这是非常好的。
5.  **缺点-** CI/CD 是少数没有大缺点的方法之一。避免它没有任何好处。然而，它确实需要一些工作来做好。确保你所有的测试都是全面的。设计一个好的 CI/CD 管道会给你的团队的运作增加一些开销，如果你的团队/项目不是很大，这可能是不值得的。然而，这些大多是次要的，不会减少好处。和其他事情一样，研究这个概念，并使用对你的情况最有用的部分。

这是 CI/CD 上的一个较长的视频。它涵盖了理想的管道，我觉得它很有价值。
